    What is git ?


    git is a version control system where files are tracked and modified accordingly easing our lives :)

  1. 3 Stages

    2. the file can be in either of the three stages

                    modified
                staged 
                commited

                therrefore the workflow is: 

                working directoery -> stagign area -> git repository



                small description :
                
                -file is modified 
                - then the files are selected to be a part of next commit 
                - which adds them to the staging ares 
                - then a commit is made which stores the changes 
                of the new version of the file

  4. The Commands - 2

      5. 
                    
    • git init ->
      •  initialises the directory .git into the current directory
                    
    • git add fullFileName or --all ->
      •  this adds files to get tracked
                    
    • git commit -m "version 1"->
      •  adds little description to the commit made
                    
    • git clone fullUrlOrPath ->
      •  clones or downloads the entire repo from an extrnal source
                    
                    any file in the repo can be in one of the two stages -- tracked or
                                                                         -- untracked 
                    
    • git status ->
      •  tells about the status of the files whether tey are being tracked or untracked
                    
    • git add fileName.extn ->
      •  adds a file to the tracking area 
                    
git add command can be used to do multiple things like tracking files , staging files and adding files to get tracked

                  
    • git status -s ->
      •  shows the status of the repo in a more simplified ways
         the notation of the simplified view -A -> the files that have been addded , M-> Modified files      
            
        
    • git diff ->
      •  gives a more simplified status of the files than the git status command
         
    • git commit ->
      •  adds a commit to the files in the staging area 
         
    • git commit -m "add the message here"->
      • commits with an added message 
         
    • git commit -a -m "this skips the staging process and directly commits the files"
      •       
         
    • git em fileName.ext ->
      •  removes the files from the staging area
         
    • git log->
      • lists the commit history
